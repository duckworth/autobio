# Project Principles

Use these as the editorial rules for the whole vault.

## Archive First

Do not start by trying to write a book. Start by building a memory archive.

The working order is:

1. capture
2. organize
3. interpret
4. write

The archive is already a success even if the final book never fully happens.

## Practical Rules

- Treat memory as perishable. If a memory keeps resurfacing, record it now.
- Prefer scenes over categories. `The day we moved in` is better than `housing`.
- Preserve first, polish later.
- Keep uncertainty visible. Do not force false confidence into memoir prose.
- Use chronology as support, not as a cage.
- Let recurring themes and turning points help determine chapter shape.
- Do not force traumatic material on a schedule. Pace it.
- A useful memoir note can be short, rough, partial, and still worth keeping.

## What Counts As Success

This project is successful if it preserves:

- real stories and scenes
- names, places, and dates that would otherwise vanish
- your point of view in your own voice
- enough structure to let later writing become honest and coherent

## Recording Bias

When choosing what to record next, prefer:

1. recurring memories
2. turning points
3. relationships that changed your life
4. places you can still describe vividly
5. transitions where the chronology or meaning is still weak

## Drafting Bias

When choosing what to write, prefer:

- one meaningful slice over exhaustive life coverage
- chapter fragments over imagined completeness
- scenes grounded in source material over smooth but unsupported prose
